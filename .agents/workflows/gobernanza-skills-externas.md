# Gobernanza de Skills Externas

## Fuentes

- `mattpocock/skills`
- `ComposioHQ/awesome-codex-skills`
- `midudev/autoskills`

## Politica

1. detectar o instalar localmente una skill no la vuelve fuente de verdad del repo
2. si una skill externa se vuelve parte estable del flujo, su uso debe documentarse en `.github/AGENTS.md`
3. si una skill externa contradice una regla interna, prevalece `.github/`

## Flujo de adopcion

1. detectar o proponer la skill
2. auditar alineacion con el stack y el dominio
3. instalar localmente
4. documentar su rol operativo
5. crear skill interna si hace falta fijar convenciones del proyecto
