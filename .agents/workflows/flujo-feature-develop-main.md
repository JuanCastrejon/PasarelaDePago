# Flujo Feature -> Develop -> Main

## Objetivo

Aplicar gitflow simple y profesional al repositorio.

## Pasos

1. actualizar `develop`
2. crear rama `feature/*`, `fix/*` o `docs/*`
3. desarrollar y validar localmente
4. abrir PR draft a `develop`
5. corregir hallazgos y checks
6. pasar a ready
7. mergear a `develop`
8. promover `develop` a `main` por PR cuando corresponda

## Reglas

- no push directo a `main`
- no push directo a `develop`
- commits atomicos y en espanol
