# Issue Tracker

## Estado actual

El tracker operativo inicial de `PasarelaDePago` es `GitHub Issues` en:

- [JuanCastrejon/PasarelaDePago](https://github.com/JuanCastrejon/PasarelaDePago)

## Regla de uso

- `GitHub Issues` es el tracker operativo por defecto para trabajo nuevo.
- El backlog fundacional historico sigue viviendo en `docs/backlog/`.
- Cuando una pieza este suficientemente estable, debe quedar reflejada tambien en `GitHub Issues`.
- Azure DevOps no reemplaza el tracker actual todavia; se adoptara despues de consolidar backlog y gobierno.

## Uso recomendado para skills

- `to-prd`, `to-issues`, `triage` y skills similares deben asumir `GitHub` como destino por defecto.
- `enrich-us` puede partir de issue o de texto, pero el flujo recomendado usa issue real cuando el cambio ya va a pasar a OpenSpec.
- Si una decision sigue en exploracion y aun no merece issue, debe permanecer en `docs/`.
