## Regla de consumo

1. Leer `CONTEXT.md` para vocabulario base.
2. Leer `indice-operativo.md` para flujo y validaciones del repo.
3. Leer `docs/dominio/` para glosario y modelo canonico.
4. Leer `docs/domain/` para reglas de negocio catalogadas.
5. Leer `docs/adr/` antes de proponer cambios estructurales.
6. Leer `docs/matriz/` para decisiones por proveedor y estados.
7. Leer `docs/requisitos/`, `docs/backlog/` y `openspec/specs/` para trabajo de producto y ejecucion.
