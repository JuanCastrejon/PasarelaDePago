# Domain Docs

## Layout

El repositorio usa layout `single-context`.

## Fuente principal

- `CONTEXT.md`

## Documentacion complementaria

- `docs/dominio/`
- `docs/adr/`
- `docs/matriz/`
- `docs/requisitos/`
- `docs/backlog/`

## Regla de consumo

1. Leer `CONTEXT.md` para vocabulario base.
2. Leer `docs/dominio/` para glosario y modelo canonico.
3. Leer `docs/adr/` antes de proponer cambios estructurales.
4. Leer `docs/matriz/` para decisiones por proveedor y estados.
5. Leer `docs/requisitos/` y `docs/backlog/` para trabajo de producto y ejecucion.
